# План развития Virage 3D Editor

Документ фиксирует результаты анализа текущего состояния приложения и поэтапный план реализации новых возможностей.

---

## Анализ текущего состояния

### Что хорошо реализовано

**Архитектура:**
- Чёткое разделение: `Editor` (состояние) → `Viewport` (рендер) → `Tools` (логика) → `UI` (интерфейс)
- EventTarget-шина между модулями — без лишних зависимостей
- Нет сборщика — статические ES-модули, работает на GitHub Pages без CI-артефактов

**Функционал:**
- Булевы операции через `three-bvh-csg` — production-ready библиотека
- Сохранение/загрузка JSON с корректной сериализацией BufferGeometry
- TransformControls + OrbitControls с правильным отключением орбиты при перетаскивании
- Тени, antialiasing, правильный `dispose()` при удалении объектов
- Клавиатурные шорткаты W/E/R/Delete/Escape

---

### Известные баги

1. **Grid snap не работает** — метод `_snap()` определён в `PrimitiveTool`, но в `create()` **никогда не вызывается**
2. **Сетка не обновляется** — при смене grid-size в выпадающем списке меняется только внутренняя переменная snap, но `GridHelper` в сцене не пересоздаётся
3. **STL export не работает** — кнопка `btn-export` есть в HTML, но обработчик нигде не зарегистрирован, `STLExporter` не подключён
4. **Undo/Redo не реализованы** — кнопки «Отмена» / «Повтор» есть в HTML, но не привязаны ни к какому коду
5. **Boolean-клик** — `this.viewport._pointerDownPos` к моменту события `pointerup` уже `null`, фильтр по дистанции dx/dy всегда равен 0 и не работает как задумано
6. **Scale в Properties Panel отсутствует** — показаны позиция и вращение, но поля для масштаба/размеров не добавлены
7. **Нет единиц мм** — поля не подписаны, связь «1 unit = 1 mm» нигде не объяснена пользователю

### Отсутствующий функционал

- Нет цвета и настройки материала объектов — все объекты один и тот же синий `#4a90d9`
- Нет импорта моделей из файлов (STL, OBJ, GLTF)
- Нет рабочей поверхности («печатного стола») с настраиваемыми размерами
- Нет мультиселекта (выделение нескольких объектов с Shift)
- Нет вытягивания по отдельным осям (только равномерный scale)
- Нет инструмента рисования плоской фигуры с последующей экструзией
- Нет списка объектов сцены
- `importmap` — не поддерживается в Safari < 16.4 и старых браузерах

---

## Поэтапный план реализации

### Этап 1 — Стабилизация основы (исправление багов)

1. **Исправить grid snap** — вызывать `_snap()` в `PrimitiveTool.create()` для начальной позиции объекта
2. **Исправить сетку** — при смене grid-size пересоздавать `GridHelper` с нужным количеством делений
3. **STL export** — подключить `STLExporter` из `three/addons/exporters/STLExporter.js`, зарегистрировать обработчик на `btn-export`. При наличии нескольких объектов — объединять все меши через `BufferGeometryUtils.mergeGeometries()` перед экспортом
4. **Scale в Properties Panel** — добавить поля Ширина / Высота / Глубина (мм), связать с `mesh.scale`
5. **Единицы мм** — подписать все числовые поля в Properties Panel

---

### Этап 1.5 — Мультиселект (Shift + клик)

Фундаментальная интерактивность — нужна до реализации операций над группой объектов.

**Изменения в `Editor`:**
- Добавить `selectedSet: Set` рядом с существующим `selected` (для обратной совместимости)
- Метод `selectAdd(mesh)` — добавить объект в `selectedSet`, выставить эмиссию
- Метод `selectRemove(mesh)` — убрать из `selectedSet`, снять эмиссию
- `select(mesh)` без Shift — как сейчас, плюс очищает `selectedSet`

**Изменения в `Viewport`:**
- Shift+клик по объекту: если объект уже в `selectedSet` → `selectRemove`; иначе → `selectAdd`
- Клик без Shift — стандартный одиночный выбор
- TransformControls аттачится к «главному» (последнему добавленному в `selectedSet`) объекту
- Остальные объекты из `selectedSet` двигаются синхронно: слушатель `change` на TransformControls вычисляет дельту и применяет её ко всем объектам группы

**Изменения в `PropertiesPanel`:**
- При мультиселекте показывать «Выбрано N объектов»
- Поля позиции/вращения/scale отображать если значения одинаковы; иначе прочерк
- Изменение значения применять ко всем объектам из `selectedSet`

**Изменения в `Toolbar`:**
- Булевы операции: если выбрано ровно 2 объекта — применять без дополнительного клика; если 3+ — применять последовательно (fold слева направо)
- Delete: удалять все объекты из `selectedSet`
- Escape: снимать мультиселект полностью
- Ctrl+A: выделить все объекты сцены

**Визуальная подсветка:**
- Каждый объект из `selectedSet` получает `emissive = 0x223344`
- При снятии выделения — `emissive = 0x000000`

---

### Этап 2 — Печатный стол + система единиц

Ключевая образовательная ориентировка: «я моделирую для 3D-печати».

1. Создать класс `PrintBed` — плоскость с сеткой, размеры задаются в мм
2. Диалог настройки стола — поля Width × Depth (мм), кнопка «Применить», сохранение в localStorage
3. Объекты при создании позиционируются на поверхности стола (Y = 0 стола)
4. Предупреждение в статусбаре при выходе объекта за границы стола
5. Отдельный визуал стола — цвет и сетка отличаются от фона сцены

---

### Этап 3 — Цвет и материал объектов

1. Color picker — `<input type="color">` в Properties Panel, обновляет `mesh.material.color`
2. Пресеты материалов — 5–6 кнопок: Пластик, Металл, Стекло, Дерево, Резина (разные комбинации `roughness` / `metalness` / базового цвета)
3. При мультиселекте — цвет и материал применяются ко всем выбранным объектам
4. Сохранение `color`, `roughness`, `metalness` в ProjectIO и восстановление при загрузке

---

### Этап 4 — Undo/Redo (Command Pattern)

1. Класс `CommandHistory` — стек команд с методами `execute(cmd)` и `undo()`
2. Команды: `AddObjectCommand`, `RemoveObjectCommand`, `TransformCommand`, `BooleanCommand`, `MaterialCommand`
3. `TransformCommand` — сохраняет snapshot позиции/вращения/scale до и после drag; при мультиселекте — один шаг отменяет движение всей группы
4. Привязка к Ctrl+Z / Ctrl+Shift+Z и кнопкам «Отмена» / «Повтор» в тулбаре
5. Лимит стека — 50 шагов

---

### Этап 5 — Импорт моделей из файлов

1. `STLLoader` из `three/addons/loaders/STLLoader.js` — поддержка бинарного и ASCII STL
2. `OBJLoader` из `three/addons/loaders/OBJLoader.js`
3. `GLTFLoader` из `three/addons/loaders/GLTFLoader.js`
4. UI — кнопка «Импорт модели» в тулбаре, `<input type="file" accept=".stl,.obj,.gltf,.glb">`
5. Нормализация импортированной модели — центрировать по bounding box, поставить на поверхность стола
6. Совместимость с ProjectIO — импортированные меши сохраняются как BufferGeometry в JSON

---

### Этап 6 — Список объектов сцены

1. Боковая панель `ScenePanel` со списком всех объектов: имя, иконка типа, кнопки «скрыть» и «удалить»
2. Мультиселект в панели — Shift+клик по имени = то же поведение, что и Shift+клик в viewport
3. Переименование — двойной клик по имени объекта
4. Синхронизация — панель обновляется при событиях `objectAdded`, `objectRemoved`, `selectionChanged`

---

### Этап 7 — Вытягивание по осям (non-uniform scale)

1. Dimension inputs в Properties Panel: Ширина / Высота / Глубина (мм)
   - Для примитивов (box, cylinder и т.д.) — пересчитывать параметры геометрии напрямую
   - Для импортированных и boolean-мешей — применять `mesh.scale` по каждой оси
2. При мультиселекте — применять одинаковый delta-scale ко всем объектам группы
3. Привязка к grid-snap при вводе числовых значений

---

### Этап 8 — Плоская фигура + экструзия

1. Режим `Draw2D` — рисование полигона кликами на XZ плоскости (вид сверху автоматически)
   - Визуализация: линии и точки в процессе рисования
   - Замыкание — двойной клик или кнопка «Готово»
2. ShapePreview — полупрозрачная заполненная фигура до экструзии
3. Диалог ExtrudeDialog — ввод высоты экструзии в мм
4. Создание меша через `THREE.ExtrudeGeometry`, размещение на столе
5. Полная интеграция — экструдированный объект участвует в transform, boolean, save/load, undo

---

### Этап 9 — Кросс-браузерная совместимость

1. Замена `importmap` на Vite — `npm run build` генерирует `dist/`, деплой через GitHub Actions
2. ES2020 target — проверить минимальную версию браузера, добавить предупреждение при необходимости
3. Тестирование в Chrome, Firefox, Safari (macOS + iOS), Edge
4. Graceful fallback при ошибке загрузки CDN — понятное сообщение пользователю
