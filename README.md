# Virage 3D Editor

Браузерный 3D-редактор для обучения детей (8–14 лет) основам 3D-моделирования с последующей 3D-печатью. Проект ориентирован на создание деталей для радиоуправляемых моделей (колёса, приводы, крепления).

**Целевая аудитория:** дети 8–14 лет, кружки робототехники и моделизма.
**Язык интерфейса:** русский.
**Платформа:** браузер (Chrome, Firefox, Safari, Edge), статический хостинг (GitHub Pages).
**Лицензия:** MIT
**Автор:** Константин Крестников ([rai220](https://github.com/rai220))

## Быстрый старт

Проект не требует сборки — это статический сайт на чистом HTML/JS/CSS.

```bash
# Клонирование
git clone https://github.com/Rai220/virage_3d_editor.git
cd virage_3d_editor

# Установка dev-зависимостей (ESLint + serve)
npm install

# Запуск локального сервера
npm run dev
```

Откройте **http://localhost:3000/** в Chrome. ES-модули требуют HTTP-сервер — `file://` протокол не работает.

### Команды

| Команда | Описание |
|---------|----------|
| `npm run dev` | Запуск dev-сервера на порту 3000 |
| `npm run lint` | Проверка кода ESLint |
| `npm test` | Запуск тестов (пока заглушка) |

## Возможности

### Стартовая сцена

При первом открытии редактор показывает три базовые фигуры — куб, сферу и цилиндр, расположенные на сетке. Это позволяет сразу начать работу: выделять, двигать, менять цвет, применять булевы операции.

### Примитивы

Пять типов 3D-примитивов доступны одним кликом прямо в верхней панели инструментов:

| Кнопка | Примитив | Геометрия |
|--------|----------|-----------|
| ◻ Куб | `BoxGeometry` | 10×10×10 мм |
| ● Сфера | `SphereGeometry` | R=5 мм, 32 сегмента |
| ⬬ Цилиндр | `CylinderGeometry` | R=5, H=10 мм |
| △ Конус | `ConeGeometry` | R=5, H=10 мм |
| ◎ Тор | `TorusGeometry` | R=5, tube=2 мм |

Каждый примитив создаётся на высоте Y=5 мм (на поверхности сетки) и может быть перемещён, повёрнут, отмасштабирован.

### Булевы операции (CSG)

Три операции для комбинирования объектов, реализованные через библиотеку **three-bvh-csg**:

| Операция | Описание | Пример |
|----------|----------|--------|
| ⊕ **Объединить** | Слияние двух тел в одно | Два пересекающихся цилиндра → крестовина |
| ⊖ **Вычесть** | Вырезание одного тела из другого | Куб − сфера → куб со сферической выемкой |
| ⊗ **Пересечь** | Только общая часть двух тел | Куб ∩ сфера → скруглённый куб |

**Как использовать:**
1. Выделите первый объект (кликните по нему)
2. Нажмите кнопку нужной операции
3. Кликните по второму объекту
4. Результат заменит оба исходных объекта

Операции работают даже при полном перекрытии объектов — система автоматически находит «другой» объект при клике.

### Трансформации

Три режима манипуляции объектами через гизмо (цветные стрелки/кольца):

| Режим | Клавиша | Описание |
|-------|---------|----------|
| Перемещение | `W` | Перетаскивание по осям X/Y/Z |
| Вращение | `E` | Поворот вокруг осей |
| Масштаб | `R` | Растяжение/сжатие по осям |

Переключение также доступно кнопками в нижней статус-панели.

### Навигация в 3D-пространстве

| Действие | Мышь | Тач-экран |
|----------|------|-----------|
| Вращение вида (орбита) | ЛКМ + перетаскивание | Один палец |
| Панорама | СКМ + перетаскивание | Два пальца |
| Зум | Колесо мыши | Щипок |
| Выделение объекта | Клик (без перетаскивания) | Тап |

Камера различает клик и перетаскивание: если мышь сместилась менее чем на 5 пикселей — это выделение, иначе — вращение.

### Предустановленные виды камеры

Быстрое переключение вида клавишами `1`–`6`:

| Клавиша | Вид |
|---------|-----|
| `1` | Спереди |
| `2` | Сзади |
| `3` | Слева |
| `4` | Справа |
| `5` | Сверху |
| `6` | Снизу |

### Панель свойств

Правая панель отображает и позволяет редактировать параметры выделенного объекта:

- **Тип** — название геометрии (box, sphere, union, subtraction и т.д.)
- **Цвет** — выбор через нативный `<input type="color">`, изменение в реальном времени
- **Позиция** — X, Y, Z в миллиметрах
- **Вращение** — X, Y, Z в градусах
- **Масштаб** — X, Y, Z (1.0 = исходный размер)

### Экспорт и импорт

| Действие | Кнопка | Горячая клавиша | Формат |
|----------|--------|-----------------|--------|
| Экспорт STL | 🖨 Экспорт STL | `Ctrl+E` | Бинарный STL (для 3D-печати) |
| Импорт STL | 📥 Импорт STL | — | STL-файл с локального диска |
| Сохранить проект | 💾 Сохранить | `Ctrl+S` | JSON (геометрия + позиции + цвета) |
| Загрузить проект | 📂 Загрузить | — | JSON-файл с локального диска |

**Формат проекта (JSON):** сохраняет для каждого объекта его геометрию (включая результаты булевых операций), позицию, поворот, масштаб и цвет. Поддерживает как параметрические геометрии (BoxGeometry, SphereGeometry и т.д.), так и произвольные буферные геометрии.

### Undo/Redo

Система отмены и повтора действий на базе паттерна **Command**:

| Действие | Горячая клавиша | Кнопка |
|----------|-----------------|--------|
| Отменить | `Ctrl+Z` | ↩ Отмена |
| Повторить | `Ctrl+Shift+Z` | ↪ Повтор |

Поддерживаемые команды:
- **AddObjectCmd** — добавление объекта (примитив, дубликат, импорт)
- **RemoveObjectCmd** — удаление объекта
- **TransformCmd** — перемещение/вращение/масштабирование через гизмо

### Удаление объектов

| Способ | Действие |
|--------|----------|
| Клавиша `Delete` | Удаляет выделенный объект |
| Клавиша `Backspace` | Альтернатива Delete |
| Кнопка 🗑 **Удалить** | Удаляет выделенный объект |
| Кнопка ✕ **Очистить всё** | Удаляет все объекты из сцены |

Удаление поддерживает Undo — `Ctrl+Z` восстанавливает удалённый объект.

### Дублирование

`Ctrl+D` — создаёт копию выделенного объекта со смещением +10 мм по оси X. Копия наследует геометрию, цвет, поворот и масштаб оригинала.

### Фокусировка

`F` — перемещает камеру так, чтобы выделенный объект оказался в центре экрана. Удобно для поиска объектов на большой сцене.

### Ключевые модули

#### `Editor.js` — центральный менеджер

Наследует `EventTarget` и управляет:
- Массивом объектов сцены (`objects[]`)
- Текущим выделением (`selected`)
- Историей команд (`CommandHistory`)
- Событиями: `objectAdded`, `objectRemoved`, `selectionChanged`, `sceneCleared`, `historyChanged`

Методы: `addObject()`, `removeObject()`, `select()`, `duplicateSelected()`, `undo()`, `redo()`, `clearScene()`, `replaceObjects()`.

#### `Viewport.js` — 3D-рендеринг

- `WebGLRenderer` с antialiasing и shadow map
- `PerspectiveCamera` (FOV 50°)
- `OrbitControls` — ЛКМ и ПКМ для вращения, СКМ для панорамы
- `TransformControls` — гизмо перемещения/вращения/масштаба
- `GridHelper` 200×200 мм + `AxesHelper` (RGB = XYZ)
- `Raycaster` для выделения объектов кликом
- Методы: `focusSelected()`, `setCameraView(name)`, `setTransformMode(mode)`

#### `Toolbar.js` — панель инструментов

Связывает HTML-кнопки с логикой. Обрабатывает:
- Создание примитивов (5 кнопок)
- Булевы операции (3 кнопки, двухшаговый процесс)
- Удаление (кнопка + Delete/Backspace)
- Очистку сцены
- Сохранение/загрузку/экспорт/импорт
- Undo/Redo (кнопки + Ctrl+Z/Shift+Z)
- Режимы трансформации (кнопки + W/E/R)
- Запись трансформаций через гизмо в историю (TransformCmd)
- Все горячие клавиши (Ctrl+D, Ctrl+S, Ctrl+E, F, 1-6, Escape)

#### `BooleanTool.js` — булевы операции

Использует `Brush` и `Evaluator` из `three-bvh-csg`. Каждая операция:
1. Клонирует геометрии и материалы исходных мешей
2. Создаёт `Brush` с правильными позициями/поворотами
3. Вычисляет результат через `evaluator.evaluate()`
4. Заменяет два исходных объекта одним результирующим

#### `CommandHistory.js` — undo/redo

Два стека: `undoStack` и `redoStack`. При выполнении новой команды — redoStack очищается. Каждая команда реализует `execute()` и `undo()`.
