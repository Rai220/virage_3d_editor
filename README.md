# Virage 3D Editor

Браузерный 3D-редактор для обучения детей (8-14 лет) основам 3D-моделирования с последующей 3D-печатью. Проект ориентирован на создание деталей для радиоуправляемых моделей (колёса, приводы, крепления).

**Целевая аудитория:** дети 8-14 лет, кружки робототехники и моделизма.
**Язык интерфейса:** русский.
**Платформа:** браузер (Chrome, Firefox, Safari, Edge), статический хостинг (GitHub Pages).

---

## Концепция

Редактор вдохновлён Autodesk 123D Design — программой, которая была стандартом для обучения 3D-моделированию до её закрытия в 2017 году. Ключевые принципы 123D Design, которые берём за основу:

- **Одна панель инструментов** вместо сложного ленточного интерфейса
- **Прямое моделирование** (Direct Modeling) — результат виден сразу, без параметрической истории
- **Примитивы + булевы операции** как основной рабочий процесс
- **Привязка граней** для интуитивной сборки деталей
- **Экспорт в STL** для 3D-печати

### Отличия от Tinkercad

Tinkercad (текущая замена 123D Design от Autodesk) слишком упрощён: нет скруглений (fillet), нет вращения профиля (revolve), нет оболочки (shell). Наш редактор занимает нишу между Tinkercad и Fusion 360 — достаточно мощный для создания реальных деталей, но достаточно простой для ребёнка.

## Целевые модели (что дети должны уметь создать)

Для проверки функциональности редактора используем конкретные детали RC-моделей:

| Деталь | Необходимые операции |
|--------|---------------------|
| **Колесо** | Цилиндр + вычитание цилиндра (ось) + круговой массив (спицы) + скругление |
| **Привод рулевой машинки к трапеции** | Прямоугольник + цилиндры (отверстия под болты) + вычитание + скругление |
| **Крепление двигателя** | Прямоугольник + отверстия + extrude из эскиза + вычитание |
| **Бампер** | Эскиз профиля + extrude + вычитание отверстий + скругление рёбер |
| **Проставка / шайба** | Цилиндр - цилиндр (кольцо) |
| **Корпусная деталь** | Box + shell (оболочка) + вычитание отверстий |

## Архитектура

### Технологический стек

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| 3D-рендеринг | **Three.js** | Стандарт для WebGL, встроенные геометрии примитивов, экспортёры |
| Манипуляция объектами | **TransformControls** (Three.js) | Готовые гизмо перемещения/вращения/масштабирования |
| Камера | **OrbitControls** (Three.js) | Орбита, панорама, зум |
| Булевы операции (CSG) | **three-bvh-csg** | В 100+ раз быстрее BSP-библиотек, нативная интеграция с Three.js |
| Валидация меша | **manifold-3d** (опционально) | Гарантия водонепроницаемости меша перед экспортом |
| Экспорт STL | **STLExporter** (Three.js) | Встроенный аддон, бинарный и ASCII форматы |
| UI-фреймворк | **Vanilla JS + HTML/CSS** | Минимум зависимостей, работа с GitHub Pages без сборки |
| Состояние | **Command Pattern + EventEmitter** | Undo/Redo, отвязка UI от 3D-движка |
| Хранение проектов | **IndexedDB** | Сохранение проектов локально в браузере |
| Хостинг | **GitHub Pages** | Бесплатно, статические файлы, CDN |

- Расстояние, Угол

## Функциональные требования

### Фаза 1 — MVP (минимально жизнеспособный продукт)

Цель: ребёнок может создать простую деталь (проставку, шайбу) и экспортировать STL.

- [ ] **Viewport**: рендер сцены, камера (орбита, панорама, зум), освещение
- [ ] **Сетка**: рабочая плоскость с миллиметровой сеткой
- [ ] **Примитивы**: куб, цилиндр, сфера (размещение кликом на сетке, ввод размеров)
- [ ] **Выделение**: клик по объекту для выделения, подсветка выбранного
- [ ] **Трансформации**: перемещение, вращение, масштабирование (гизмо TransformControls)
- [ ] **Привязка к сетке**: snap с настраиваемым шагом (1мм, 5мм, 10мм)
- [ ] **Булевы операции**: объединение, вычитание, пересечение (three-bvh-csg)
- [ ] **Экспорт STL**: скачивание файла через браузер (бинарный STL)
- [ ] **Undo/Redo**: отмена и повтор действий (Ctrl+Z / Ctrl+Shift+Z)
- [ ] **Панель свойств**: отображение и редактирование позиции, размеров
- [ ] **Русский интерфейс**: все надписи, подсказки, сообщения на русском

### Фаза 2 — Полноценное моделирование

Цель: ребёнок может создать колесо или привод рулевой машинки.

- [ ] **Дополнительные примитивы**: конус, тор, клин
- [ ] **Эскизы**: рисование 2D-профилей на плоскостях (прямоугольник, круг, многоугольник, линия)
- [ ] **Выдавливание (Extrude)**: создание 3D-тела из 2D-эскиза
- [ ] **Вращение профиля (Revolve)**: создание тела вращения из эскиза
- [ ] **Скругление (Fillet)**: скругление выбранных рёбер с заданным радиусом
- [ ] **Фаска (Chamfer)**: срез ребра под углом
- [ ] **Оболочка (Shell)**: создание полого объекта с заданной толщиной стенки
- [ ] **Круговой массив**: повтор элемента вокруг оси (для спиц колеса)
- [ ] **Линейный массив**: повтор элемента вдоль направления (для отверстий крепления)
- [ ] **Зеркало**: зеркальное отражение относительно плоскости
- [ ] **Измерение**: расстояние между точками/гранями, угол
- [ ] **ViewCube**: интерактивный куб ориентации для быстрой смены вида
- [ ] **Привязка граней (Snap)**: выравнивание объектов грань-к-грани

### Фаза 3 — Продвинутые функции и UX

Цель: удобная работа, библиотека деталей, учебные материалы.

- [ ] **Библиотека деталей**: готовые детали RC-моделей (колесо, рама, крепление)
- [ ] **Сохранение проектов**: IndexedDB, экспорт/импорт проекта в JSON
- [ ] **Импорт STL**: загрузка существующих моделей для модификации
- [ ] **Группировка**: объединение объектов в группу для совместного перемещения
- [ ] **Цвет/материал**: визуальный цвет объектов (для различения деталей)
- [ ] **Ортографическая проекция**: переключение перспектива/ортография
- [ ] **Горячие клавиши**: полный набор (Del — удалить, Ctrl+D — дублировать, и т.д.)
- [ ] **Контекстное меню**: правый клик на объекте
- [ ] **Обучающие подсказки**: интерактивные подсказки при первом запуске
- [ ] **Адаптивность**: корректная работа на планшетах (тач-управление)
- [ ] **Тёмная/светлая тема**: переключение для комфорта

## Дизайн-принципы

### Для детей 8-14 лет

1. **Крупные кнопки и иконки** — минимум 44x44px (рекомендация Apple для touch targets)
2. **Иконки + текст** — не только иконки, чтобы было понятно без угадывания
3. **Немедленная обратная связь** — каждое действие визуально подтверждается
4. **Ограничение выбора** — 6-8 инструментов видно сразу, остальные в подменю
5. **Безопасность ошибок** — Undo всегда доступен, нет необратимых действий
6. **Подсказки при наведении** — tooltip на русском языке для каждого инструмента
7. **Цветовое кодирование** — оси X(красный), Y(зелёный), Z(синий) как в стандартных CAD
8. **Отсутствие регистрации** — открыл сайт, начал работать

### Визуальный стиль

- Светлый фон рабочей области (#f0f0f0)
- Тёмно-серая панель инструментов (#2d2d2d)
- Акцентный цвет: синий (#4a90d9) — выделение, активные элементы
- Шрифт: системный sans-serif, размер не менее 14px
- Иконки: монохромные SVG, белые на тёмном фоне панели

## Требования к экспорту STL

Для корректной 3D-печати экспортированный STL должен:

1. **Водонепроницаемость (manifold)** — меш без дыр и самопересечений
2. **Правильная ориентация нормалей** — все нормали направлены наружу
3. **Единицы измерения — миллиметры** — стандарт для слайсеров (Cura, PrusaSlicer)
4. **Бинарный формат** — компактный размер файла
5. **Имя файла** — транслитерация или латиница для совместимости
