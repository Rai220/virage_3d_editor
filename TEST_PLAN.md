# План тестирования Virage 3D Editor

Этот документ описывает ручные тесты, которые необходимо проходить после каждого значительного изменения в коде. Каждый тест имеет чёткие шаги и ожидаемый результат.

---

## 1. Запуск и стартовая сцена

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| 1.1 | Открыть `http://localhost:3000/`, дождаться загрузки | Страница загружается без ошибок в консоли |
| 1.2 | Проверить стартовую сцену | На сцене три фигуры: куб (слева), сфера (центр), цилиндр (справа) |
| 1.3 | Проверить панель инструментов | Видны кнопки: Virage 3D, 5 примитивов, 3 булевых, Удалить, Очистить всё, Сохранить, Загрузить, Экспорт STL, Импорт STL, Отмена, Повтор |
| 1.4 | Проверить статус-бар | Видны: Перемещение/Вращение/Масштаб, сетка 5мм, статус, ссылки автора |
| 1.5 | Проверить панель свойств | Текст «Выберите объект для редактирования» |

## 3. Создание примитивов

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| 3.1 | Нажать «Куб» | Куб появляется в сцене, статус «Создан: Куб» |
| 3.2 | Нажать «Сфера» | Сфера появляется |
| 3.3 | Нажать «Цилиндр» | Цилиндр появляется |
| 3.4 | Нажать «Конус» | Конус появляется |
| 3.5 | Нажать «Тор» | Тор появляется |

## 4. Выделение и трансформации

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| 4.1 | Клик по объекту | Объект выделяется, гизмо появляется, панель свойств обновляется |
| 4.2 | Клик на пустом месте | Выделение снимается |
| 4.3 | Перетащить стрелку гизмо (режим перемещения) | Объект перемещается по оси |
| 4.4 | Нажать `E`, перетащить кольцо | Объект вращается |
| 4.5 | Нажать `R`, перетащить ручку | Объект масштабируется |
| 4.6 | Нажать `W` | Возврат в режим перемещения |
| 4.7 | Нажать кнопки «Перемещение» / «Вращение» / «Масштаб» в статус-баре | Соответствующий режим активируется |

## 5. Булевы операции

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| 5.1 | Создать куб и сферу (перекрывающиеся) | Два объекта в сцене |
| 5.2 | Выделить куб, нажать «Объединить», кликнуть по сфере | Два объекта заменяются одним (union), статус «Объединение выполнено» |
| 5.3 | `Ctrl+Z` | Оба исходных объекта восстанавливаются |
| 5.4 | `Ctrl+Shift+Z` | Результат объединения снова появляется |
| 5.5 | Создать куб и сферу, выделить куб, нажать «Вычесть», кликнуть по сфере | Куб со сферической выемкой, статус «Вычитание выполнено» |
| 5.6 | `Ctrl+Z` | Оба исходных объекта восстанавливаются |
| 5.7 | Создать куб и сферу, выделить куб, нажать «Пересечь», кликнуть по сфере | Только пересечение двух тел, статус «Пересечение выполнено» |
| 5.8 | `Ctrl+Z` | Оба исходных объекта восстанавливаются |

## 6. Удаление

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| 6.1 | Выделить объект, нажать кнопку 🗑 «Удалить» | Объект удалён, статус «Объект удалён» |

| 6.3 | Выделить объект, нажать `Backspace` | Объект удалён |

## 8. Undo/Redo

| # | Шаг | Ожидаемый результат |
|---|-----|---------------------|
| 8.1 | Создать объект, нажать `Ctrl+Z` | Объект исчезает |

## Чек-лист быстрой проверки

Минимальный набор тестов для прогона после каждого значительного изменения:

- [ ] Страница загружается, 3 стартовых объекта на месте
- [ ] Создание примитива (любого) работает
- [ ] Выделение объекта кликом работает
- [ ] Перемещение объекта гизмо работает
- [ ] Вращение камеры (ЛКМ + drag) работает
- [ ] Булева операция (вычитание) выполняется
- [ ] `Ctrl+Z` отменяет булеву операцию
- [ ] `Ctrl+Z` отменяет создание объекта
- [ ] Удаление объекта (Delete) работает
- [ ] Сохранение (`Ctrl+S`) скачивает JSON
- [ ] Экспорт STL скачивает файл
- [ ] `npm run lint` проходит без ошибок
